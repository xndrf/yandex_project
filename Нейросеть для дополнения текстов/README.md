##  Описание проекта

Проект посвящен решению задачи **автодополнения текстов** — созданию нейросети, которая на основе начала фразы предсказывает её продолжение. 
Иными словами подготовка нейросети для:

- Клавиатур и мессенджеров
- Поисковых систем
- Голосовых ассистентов
- Email-композиторов
- Генерации историй


### Задачи:
1. Взять датасет с короткими постами
2. Реализовать и обучить **LSTM модель**
3. Замерить качество
4. Сравнить с предобученным **DistilGPT2**
5. Дать рекомендации разработчикам

## Задача машинного обучения 

Автодополнение на языке ML - многократное предсказание следующего токена**



### Данные
- **Источник**: датасет Sentiment140 (твиты)
- **Объем данных**: 1,600,000+ твитов
- **Данные на обучения**: 1,000,000 твитов
- **Разделение**:
  - Train: 80% (800,000)
  - Validation: 10% (100,000)
  - Test: 10% (100,000)

### Метрики
- **ROUGE-1** — совпадение униграмм (отдельных слов)
- **ROUGE-2** — совпадение биграмм (пар слов)

### Архитектуры

#### LSTM (наша модель)
- Embedding: 256 dim
- LSTM слои: 2 слоя по 512 hidden dim
- Dropout: 0.3
- Параметры: ~10M

#### DistilGPT2 (для сравнения)
- Предобученный трансформер
- Параметры: 82M
- "Дистиллированная" версия GPT2

## **КЛЮЧЕВЫЕ ВЫВОДЫ ПО ПРОЕКТУ**

  - Исходный датасет был достаточно чистым
  - Большинство твитов действительно на английском языке
  - Примененная чистка была щадящей

#### Выполненные этапы подготовки данных:
- Приведение к нижнему регистру
- Удаление URL и упоминаний

**Результат**: Подготовлен чистый датасет англоязычных твитов, готовый для обучения моделей.



### 2. **СРАВНЕНИЕ МОДЕЛЕЙ**

#### Количественные метрики (ROUGE)

| Модель | ROUGE-1 | ROUGE-2 | Параметры |
|--------|---------|---------|-----------|
| **LSTM (обученная)** | **0.0845** | **0.0209** | ~10M |
| DistilGPT2 | 0.0664 | 0.0000 | 82M |

**Ключевые наблюдения:**
- **LSTM превзошла DistilGPT2 на 27% по ROUGE-1**
- **LSTM улавливает биграммы** (ROUGE-2 = 0.0209), в то время как GPT2 показывает **нулевой результат**
- **Абсолютные значения метрик низкие** у обеих моделей — это сигнал о проблеме с данными, а не с архитектурами.

**Главный вывод:** Низкие метрики и проблемы с качеством генерации связаны не с архитектурами моделей, а с **ограничениями обучающих данных**. Модель научилась генерировать англоязычный текст, но с потерей контекста, эмоциональными скачками и незаконченностью.
